# Bias Calibration Tool

## 要件定義書 + 基本設計

### 1. システム概要

本システムは、カセットデッキの調整を支援する macOS 向け GUI アプリケーションであり、以下の機能を提供する。

- バイアス調整（Noise + Tone）
- 対話型ガイド
- Wow & Flutter 測定
- テスト音源生成・書き出し

### 2. システム目的

- スペクトラム解析器なしでも調整できる体験を提供する
- 初心者でも段階的に調整できるようにする
- オーディオ IF を前提とした実用的な精度を確保する

### 3. スコープ

#### 対象

- カセットデッキ（Type I / II / IV）
- オーディオ IF のライン入力
- オーディオ IF 経由の外部オーディオ入出力

#### 接続前提

- 本アプリは、カセットデッキの録音系または再生系をオーディオ IF のライン入力へ接続して使用することを前提とする
- マイク入力は対象外とする
- オーディオ IF 側で入力ゲインを固定できることを前提とする
- 可能であれば、アプリの信号出力も同一オーディオ IF のライン出力を使用する

#### 非対象

- EQ 調整
- ノイズリダクション調整
- 自動バイアス制御
- マイク入力を用いた測定

### 7. 機能仕様

#### 7.1 Noise Calibration（粗調整）

##### 目的

- 周波数特性の補正方向を把握する

##### 入力

- ホワイトノイズ

##### 処理

```text
高域平均（8kHz ～ 12kHz）
中域平均（1kHz ～ 4kHz）
差分算出
```

##### 出力

- バイアスメーター
- 状態表示（OVER / OK / UNDER）

##### 判定

| 値 | 状態 |
| --- | --- |
| > +0.5 dB | UNDER |
| -0.5 dB ～ +0.5 dB | OK |
| < -0.5 dB | OVER |

##### 判定値に関する注意

- 上記の `±0.5 dB` は固定の絶対精度を保証する値ではなく、実運用上の初期目安である
- 判定は、使用するカセットデッキ、テープ種別、ヘッド状態、録再レベル、オーディオ IF の周波数特性および入力ゲイン設定の影響を受ける
- そのため、最終的な判定閾値はテープ種別ごとに再調整可能な設計とする
- UI 上では、将来的に `Type I / II / IV` ごとの推奨閾値プリセットを持てる構成が望ましい

#### 7.2 Tone Calibration（仕上げ）

##### 目的

- 高域トーンのピーク位置から最適バイアス近傍を求める

##### 入力

- 10kHz サイン波

##### 処理

```text
出力レベル測定
最大値検出
現在値との差分算出
```

##### 出力

- レベルメーター
- Peak Hold
- 状態表示

##### 判定

| 条件 | 状態 |
| --- | --- |
| peak 付近 | MAX |
| peak -0.5 dB ～ -1.5 dB | 推奨 |
| peak より上 | UNDER |
| peak より下 | OVER |

#### 7.3 Guided Mode

##### 目的

- 初心者でも順番どおりに調整できるようにする

##### フロー

```text
1. 準備
2. Noise Calibration
3. Tone Calibration
4. 完了
```

##### UI

- ステップ形式
- メッセージ表示
- ボタン遷移

#### 7.4 Wow & Flutter

##### 入力

- 3kHz トーン

##### 処理

```text
瞬時周波数検出
変動量算出
RMS 算出
```

##### 出力

| 値 | 評価 |
| --- | --- |
| < 0.05% | EXCELLENT |
| < 0.1% | GOOD |
| < 0.2% | OK |
| >= 0.2% | BAD |

##### 注意事項

- 本機能は簡易測定機能であり、JIS / DIN / IEC などの正式規格準拠を保証するものではない
- 実用上の比較用途には利用できるが、測定規格に基づく公称値確認には別途専用測定器が望ましい

#### 7.5 Signal Export

##### 目的

##### 出力ファイル

##### 仕様

| ビット深度 | 16bit |

### 9. アルゴリズム設計

#### 9.1 Noise

```text
normalized = high_band - mid_band
bias_error = tape - source
```

#### 9.2 Tone

```text
peak = max(level)
delta = current - peak
```

#### 9.3 Wow & Flutter

```text
Δf / f → RMS
```

### 10. レベル基準合わせ

`bias_error = tape - source` を成立させるには、ソース側と録再側のレベル基準が揃っている必要がある。本システムでは、少なくとも以下を前提条件またはセットアップ手順として持つ。

#### 10.1 必要条件

- 信号出力に使用するオーディオ IF と、信号入力に使用するオーディオ IF を固定する
- 入出力のサンプルレートを一致させる
- オーディオ IF の入出力ゲインを固定し、測定中に変更しない
- OS やドライバの自動音量補正機能を無効化する
- カセットデッキ側の Dolby、リミッター、音質補正機能は無効化する

#### 10.2 推奨セットアップ手順

1. オーディオ IF の出力に基準信号を出す
2. 可能であればループバック接続して、アプリ内で入出力レベル差を確認する
3. 基準トーン入力時に 0 dB の基準位置を決め、アプリ側で補正値として保存する
4. その補正値を `source reference offset` として保持し、Noise / Tone 判定時に常に適用する

#### 10.3 記録すべき設定情報

- 使用オーディオ IF 名称
- 入力チャンネル
- 出力チャンネル
- サンプルレート
- ビット深度
- 入力ゲイン設定
- 出力ゲイン設定
- テープ種別
- デッキ名
- 基準オフセット値

#### 10.4 実装上の扱い

- 初回起動時または設定変更時に、簡易キャリブレーション画面を用意する
- キャリブレーション未実施時は `参考値` として扱い、厳密判定を避ける
- キャリブレーション実施後のみ `OK / OVER / UNDER` 判定を確定表示する設計が望ましい

### 11. macOS / オーディオ経路の注意事項

macOS 側のオーディオ経路に自動補正やサンプルレート変換が入ると、測定結果が不安定になる。以下を仕様上の注意事項として明記する。

#### 11.1 無効化または回避すべき要素

- 自動ゲイン調整（AGC）
- 音声強調、ノイズ抑制、会議アプリ由来の音声処理
- OS またはドライバによるサンプルレート変換
- ミキサーアプリでのインサートエフェクトや DSP

#### 11.2 アプリ上の制約

- 対応入力デバイスは、固定サンプルレート・固定ゲインで扱えるオーディオ IF を前提とする
- 入力デバイスがマイク系として認識される場合は、警告表示を行う
- アプリ起動時に選択中デバイスのサンプルレートとチャンネル構成を表示する
- アプリ側の要求サンプルレートと実デバイスの設定が一致しない場合は測定開始を抑止する

#### 11.3 ユーザー向け注意表示

- 他の音声処理アプリを終了してから測定すること
- Audio MIDI 設定でサンプルレートを確認すること
- macOS の入力音量やデバイス設定を測定途中で変更しないこと

### 17. 完成定義

## 最終まとめ

- Noise で方向を掴む
- Tone で最終位置を決める
- GUI で直感的に操作できる
- 精度確保にはオーディオ IF と事前キャリブレーションが必須である
